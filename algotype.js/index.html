Algotype.js - typesetting pseudocode in HTML files

Algotype.js?

This vanilla JavaScript library allows users to typeset pseudocode with custom XML tags and MathJax in HTML documents. The motivation was that people (for the most part) don't need to take representation of pseudocode into account, but rather concentrate on the algorithm logic.

Including Algotype.js

In order to load the Algotype.js library, all you need is: 

<link rel="stylesheet" type="text/css" href="algotype.css">
<link rel="stylesheet" type="text/css" href="algotype_my_config.css">
<script src="algotype.js"></script>
in the <head> of the HTML file. Feel free, however, to get rid of algotype_my_config.css and add all the custom CSS rules straight into algotype.css.

Dependencies

Algotype.js implicitly loads MathJax library for writing TeX.

alg-algorithm

Summary

The tag alg-algorithm defines an entire algorithm. Naturally, it may have all inner elements except alg-algorithm.

Synopsis

<alg-algorithm name="Algorithm-Name" parameters="x A R" comment="Algorithm header comment">
...
</alg-algorithm>
The above will be rendered as follows

namethe name of the algorithmrequired
parametersthe list of algorithm's parametersrequired
commentthe algorithm commentoptional

Notes

  • Make sure that you wrap the element <alg-algorithm>...</alg-algorithm>; into a, say, <div> that does not contain any other elements. Otherwise, it will be placed as the very last element of its parent element even if the algorithm is not the last element of the parent.
  • The parameter list accepts several formats. The above parameter list $(x, A, R)$ may be specified in parameters attribute as
      x A R
      x, A, R
      (x A R)
      (x, A, R)
    • You can also omit some commas in the above lists, or omit one or both parentheses.

alg-break

Summary

The tag alg-break typesets a break statement. This element may not have inner elements.

Synopsis

<alg-break id="break-id" comment="Break comment" comment-id="break-comment-id">Label<alg-break>
The above will be rendered as follows at line 1:

Label

idthe break statement IDoptional
commentthe break statement commentoptional
comment-idthe break statement comment IDoptional

Notes

  • The token Label is optional and is used most often for breaking out of an outer loop.

alg-return

Summary

The alg-return tag typesets return statements.

Synopsis

<alg-return id="return-id" comment="Return comment" comment-id="return-comment-id">$1 \; + $ Aux-Procedure$(x, y, Q)&doll$</alg-return>
The above will be rendered as follows at line 1:

$1 \; +$ Aux-Procedure$(x, y, Q)$

id the return statement ID optional
comment the return statement comment optional
comment-id the return statement comment ID optional

Notes

In the alg-return element, all text not delimeted by $ TeX delimeters is considered to be a function call and is thus typeset using small-caps font just like the algorithm name.
